# Token Porter — Figma Plugin

Import and export Figma Variables ZIPs generated by the Figma Variables Tokens Generator skill.

## Installation (Local Development)

1. Open Figma Desktop
2. Menu → Plugins → Development → Import plugin from manifest
3. Select the `manifest.json` from this folder
4. The plugin will appear under Plugins → Development → Token Porter

## Import

1. Run the plugin → Import tab
2. Drop or browse for your `design-tokens.zip`
3. Review the collection preview (names, modes, token counts)
4. Click **Import All Collections**

Collections are created in the correct order with correct default modes:
- Theme: light first (default), dark second
- Responsive: mobile first (default), tablet, desktop
- Density: comfortable first (default), compact, spacious

After import you'll see a per-collection token count and any unresolved aliases.

## Export

1. Open a Figma file with local Variable collections
2. Run the plugin → Export tab
3. Collections are listed in standard import order
4. Click **Export as ZIP**
5. Download `design-tokens.zip` — ready to re-import or use with Style Dictionary

## ZIP Structure (Import format)

```
design-tokens.zip
├── 1. Primitives/
│   └── primitives.tokens.json
├── 2. Theme/
│   ├── light.tokens.json
│   └── dark.tokens.json
├── 3. Responsive/
│   ├── mobile.tokens.json
│   ├── tablet.tokens.json
│   └── desktop.tokens.json
...
```

## Notes

- Requires internet connection on first use (loads JSZip from CDN)
- Re-importing does not replace existing collections — Figma creates duplicates. Delete old ones first.
- Scopes and codeSyntax metadata are not applied on import (Figma API limitation). These are visual-only metadata in the JSON.
